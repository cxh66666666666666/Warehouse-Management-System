## 项目分层职责说明

### 1. controllers（请求接入层）
负责接收 HTTP 请求，进行参数校验，将整理好的参数传递给 service 层处理。
当前状态：部分 controller 代码中包含了本应属于 service 层的数据库处理代码，需要逐步重构。

### 2. services（业务编排层）
负责处理核心业务逻辑，通过数据结构体与网络层完全解耦。需要完整考虑业务处理流程，而不仅仅是简单的增删改查。

### 3. models（数据映射层）
提供数据库表结构的 C++ 对象映射。ORM 类（如 `Product`, `Inventory` 等）定义了表字段与 C++ 类型的对应关系。
**重要**：models 层**仅负责数据结构映射**，真正查询数据仍需要通过 `Mapper`、执行 SQL 或调用 ORM 查询接口。不要自行修改由框架自动生成的 model 文件。

### 4. test（测试层）
负责测试项目是否正常运行，模拟各种 error 发生时系统的反应。

### 5. qt（前端层）
独立的前端代码，拥有独立的 cmake 配置，与后端代码隔离。
