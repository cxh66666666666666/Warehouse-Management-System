    controllers:在整个项目中作为服务员的角色，根据请求路径和操作类型接收前端传来的参数，校验参数（如字段完整性，字段类型适配等），将参数整理成struct后调用services层处理任务，但目前我的controllers代码包含了很多数据库处理代码，这些代码本应该出现在services，后续需要修改。
    services：处理业务逻辑，并且通过struct完全与网络解耦，需要考虑到整个业务处理而不是简单的增删改查
    models：mysql的c++镜像，drogon自动将数据库翻译为c++的类，这样就不需要再在代码中查询数据库，只需要简单调用类函数就可以拿到字段，绝对不要自己修改models的文件
    test：负责测试项目是否正常运行，模拟各种error发生时系统的反应
    qt：前端代码，拥有独立的cmakelist，不会污染后端代码
