### 1. 端口与监听配置 (`listeners` 节点)
这个节点用于配置服务器监听的端口和网络地址。
*   `address`: 绑定的 IP 地址。通常设为 `"0.0.0.0"`，表示接受来自任何网卡的请求；如果设为 `"127.0.0.1"`，则只能本机访问。
*   `port`: 服务监听端口（例如 `8080`、`80`）。
*   `https`: 是否启用 HTTPS 加密（`true` 或 `false`）。启用时必须配合配置 `cert`（SSL 证书路径）和 `key`（SSL 私钥路径）。

---

### 2. 线程数与并发配置 (`app` 节点)
这个节点决定了服务器能同时处理的并发压力。
*   `number_of_threads`: **核心工作线程数**。
    *   如果设为 `0`，Drogon 会自动获取服务器的 **CPU 逻辑核心数**作为线程数，这是生产环境推荐配置。
*   `max_connections`: 服务器允许保持的**最大并发连接总数**（例如 `100000`），超过会被拒绝。
*   `max_connections_per_ip`: 限制**单个 IP** 的最大连接数（例如 `0` 表示不限制）。
*   `idle_connection_timeout`: 空闲连接超时时间（秒），超过则自动断开。

---

### 3. 数据库配置 (`db_clients` 节点)
配置应用连接的底层数据库。Drogon 支持同时连接多个不同的数据库。
*   `rdbms`: 数据库类型，例如 `"mysql"`、`"postgresql"`、`"sqlite3"`。
*   `host` / `port`: 数据库服务器的 IP 地址和端口。
*   `dbname` / `user` / `passwd`: 数据库名、登录账号、密码。
*   `is_fast`: **高性能异步模式开关**。
    *   设为 `true` 时性能大幅提升，但代码**不能调用任何同步阻塞 SQL 接口**。
*   `number_of_connections`: 数据库连接池大小。
    *   若 `is_fast` 为 `true`，代表**每个 IO 线程**的连接数；若为 `false`，则代表全局总连接数。
*   `timeout`: SQL 查询的超时时间（秒），负数（如 `-1.0`）表示永不超时。

---

### 4. 日志级别与输出配置 (`app.log` 节点)
控制系统运行时的日志输出内容。
*   `log_level`: 日志输出级别，从低到高可选：`"TRACE"`、`"DEBUG"`、`"INFO"`、`"WARN"`、`"ERROR"`、`"FATAL"`。
    *   开发时设为 `"DEBUG"`，可看到详细的 SQL 语句；**生产环境应改为 `"INFO"` 或 `"WARN"`**。
*   `logfile_base_name`: 日志文件的文件名前缀（不配置则只输出到控制台）。
*   `log_size_limit`: 单个日志文件最大体积（字节），超过会自动切分新文件。
*   `max_files`: 保留的旧日志文件最大数量，设为 `0` 表示永不删除。
*   `display_local_time`: 是否显示本地时区时间（`true`），否则打印 UTC 时间。
