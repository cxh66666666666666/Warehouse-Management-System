### 1. 端口与监听配置 (`listeners` 节点)
这个节点用于告诉服务器应该在哪些网络端口上“竖起耳朵”听请求。你可以配置多个监听器（比如同时监听 HTTP 和 HTTPS）。
*   `address`: 绑定的 IP 地址。通常设为 `"0.0.0.0"`，表示允许来自任何网卡/局域网/公网的请求访问。如果设为 `"127.0.0.1"`，则只能在本机访问。
*   `port`: 服务监听端口（例如 `8080`、`80`）。
*   `https`: 是否启用 HTTPS 加密（`true` 或 `false`）。如果设为 `true`，必须配合配置 `cert` (SSL 证书路径) 和 `key` (SSL 私钥路径)。

---

### 2. 线程数与并发配置 (`app` 节点)
这个节点决定了你的服务器能同时扛住多少并发压力。
*   `threads_num` (或 `number_of_threads`): **核心工作线程数**。
    *   **关键用法**：如果你把它设置为 `0`，Drogon 会自动获取你服务器的 **CPU 逻辑核心数**，并启动对应数量的线程。这是生产环境最推荐的配置，能最大化榨干多核 CPU 的性能。
*   `max_connections`: 服务器允许保持的**最大并发连接总数**（例如 `100000`）。超过这个数的连接会被直接拒绝，用于保护服务器不被压垮。
*   `max_connections_per_ip`: 限制**单个 IP** 能够建立的最大连接数（例如 `0` 表示不限制，设为 `10` 可以防止简单的恶意刷接口）。
*   `idle_connection_timeout`: 空闲连接超时时间（例如 `60` 秒）。如果一个客户端连上后 60 秒都不发数据，服务器会主动断开它以释放资源。

### 3. 数据库配置 (`db_clients` 节点)
这个节点用于配置应用连接的底层数据库。Drogon 支持同时连接多个不同的数据库。
*   `rdbms`: 数据库类型。例如 `"mysql"`、`"postgresql"`、`"sqlite3"`。
*   `host` / `port`: 数据库服务器的 IP 地址和端口（例如 `"127.0.0.1"` 和 `3306`）。
*   `dbname` / `user` / `passwd`: 数据库名、登录账号、密码。
*   `is_fast`: **高性能异步模式开关**。
    *   如果设为 `true`，数据库客户端的性能会极大幅度提升，但代价是你的 C++ 代码**绝对不能调用任何同步的阻塞 SQL 接口**（只能用异步回调或协程 `co_await`）。
*   `connection_number` (或 `number_of_connections`): 数据库连接池大小。
    *   **注意**：它的含义受 `is_fast` 影响。如果 `is_fast` 为 `true`，这代表**每个 IO 线程**的连接数；如果为 `false`，则代表所有线程共享的**全局总连接数**。
*   `timeout`: SQL 查询的超时时间（单位：秒）。设置为负数（如 `-1.0`）表示永不超时。

---

### 4. 日志级别与输出配置 (`app.log` 节点)
控制系统运行时的黑框框（或日志文件）里会打印什么信息。
*   `log_level`: 日志输出级别。
    *   可选值从低到高：`"TRACE"`, `"DEBUG"`, `"INFO"`, `"WARN"`, `"ERROR"`, `"FATAL"`。
    *   开发时通常设为 `"DEBUG"`，能看到详细的 SQL 语句和框架底层的运转细节；**上线生产环境时要改为 `"INFO"` 或 `"WARN"`**，否则海量的日志会拖慢服务器性能并撑爆硬盘。
*   `logfile_base_name`: 日志文件的文件名前缀（如果不配置，默认只输出到控制台）。
*   `log_size_limit`: 单个日志文件的最大体积（单位：字节）。例如 `100000000`（约 100MB），超过这个大小后，系统会自动切分出一个新的日志文件（日志轮转）。
*   `max_files`: 保留的旧日志文件最大数量（例如 `10`）。设为 `0` 表示永远不删除旧文件。
*   `display_local_time`: 是否将日志时间显示为本地时区时间（`true`）。若设为 `false` 则打印 UTC 标准时间。
